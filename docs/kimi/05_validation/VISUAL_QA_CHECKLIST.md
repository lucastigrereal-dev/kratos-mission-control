# Checklist de QA Visual

Use este checklist após implementar microfases de visual.  Marque cada item conforme:

| Item | OK | Ajuste Leve | Divergência | Bloqueante |
|---|---|---|---|---|
| **Missão Atual** aparece claramente na tela inicial |  |  |  |  |
| **Próxima Ação** é visível em 10 segundos |  |  |  |  |
| **Paleta** segue os tokens de oceano, céu, ilhas e energia |  |  |  |  |
| **Glassmorphism** consistente nos painéis |  |  |  |  |
| **HUD** não polui e não lembra SaaS |  |  |  |  |
| **Mapa** parece mundo vivo (não círculos/bolhas) |  |  |  |  |
| **Ilhas** têm profundidade e glow adequados |  |  |  |  |
| **Aurora** está presente, legível e não invasiva |  |  |  |  |
| **Dock Inferior** exibe missão, próxima ação e botão |  |  |  |  |
| **SourceBadge** aparece em todos os dados reais |  |  |  |  |
| **Preferência de Motion** respeitada (prefers-reduced-motion) |  |  |  |  |
| **Sem artefatos** de CSS quebrado (bordas, sombras) |  |  |  |  |

Para cada item marcado como “Ajuste”, “Divergência” ou “Bloqueante”, explicar no relatório qual o problema e em qual microfase será corrigido.
